# Network Traffic Analysis Using Wireshark

## Project Overview

This project documents the process of analyzing network traffic using Wireshark to understand communication between devices, identify protocols, and recognize potential security concerns.

## Environment

Type:
Training / Simulated Network Analysis Exercise

Tool:
Wireshark

Purpose:
To develop practical understanding of network protocols, packet analysis, and security monitoring concepts.

---

# Objectives

The objectives of this exercise were to:

- Understand network packet structure
- Identify common network protocols
- Analyze communication patterns
- Recognize potential security indicators

---

# Networking Concepts Applied

## TCP/IP

Reviewed how devices communicate using:

- IP addressing
- Transport protocols
- Network communication processes

## DNS

Analyzed the role of DNS in converting domain names into IP addresses.

## HTTP/HTTPS

Reviewed how web communication occurs and the differences between encrypted and unencrypted traffic.

---

# Analysis Process

The analysis workflow includes:

1. Capturing network traffic
2. Filtering packets based on protocols
3. Reviewing source and destination addresses
4. Identifying communication patterns
5. Documenting observations

# Security Observations

During network analysis, important indicators to review include:

- Unexpected communication with unknown hosts
- Unusual traffic patterns
- Repeated connection attempts
- Clear-text transmission of sensitive information
